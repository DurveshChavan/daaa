# Literature Review Summary Table

> AI-Enabled Resource Detection & Allocation Using Random Forest

---

## Core Papers

| # | Paper Name | Objective | Methodology | Result | Limitations |
|---|---|---|---|---|---|
| 1 | Intelligent Resource Allocation in IoT Using RF and Clustering (Ahmad et al., 2025) | To develop an intelligent resource allocation framework for IoT networks | Combined K-Means clustering (grouping IoT devices by energy, bandwidth, and compute demand) with Random Forest classification to predict optimal resource tiers | 94% prediction accuracy; 20% energy reduction; 10% decrease in response time vs. conventional allocation | Single-domain focus (IoT only); limited scalability analysis for large-scale deployments; no formal algorithmic complexity analysis |
| 2 | AI-Augmented Kernel for Dynamic Resource Utilization (Gupta & Sharma, 2025) | To embed AI-based resource allocation within the Linux kernel for real-time CPU/memory management in virtualized environments | RF model integrated at kernel level using cgroups for resource isolation and psutil for metric collection; classifies resource demand in real-time | 99.34% classification accuracy for resource demand prediction; demonstrated RF inference is lightweight enough for OS-level scheduling | Restricted to CPU/memory metrics; kernel-level integration limits portability across different OS platforms; no cross-domain validation |
| 3 | GA-RF Hybrid for HPC Job Allocation (Martinez et al., 2024) | To optimize RF hyperparameters using Genetic Algorithms for improved HPC job scheduling in data centers | GA optimized RF hyperparameters (number of estimators, tree depth, feature subsampling ratio); evaluated on ISPES benchmark traces | 12.4% improvement in makespan; 8.7% improvement in resource utilization over standalone RF | Focused only on HPC workloads; GA optimization adds computational overhead; no energy efficiency analysis; limited to batch scheduling scenarios |
| 4 | Random Forest-Based Cloud Task Scheduling (Chen et al., 2018) | To classify cloud tasks by resource intensity and match them to optimal VM configurations | RF classifier categorized tasks as CPU-bound, memory-bound, or I/O-bound; compared with decision trees and SVM for multi-class classification | 91.3% accuracy; RF outperformed decision trees and SVM in multi-class task classification | Foundational/early work with dated evaluation; default RF hyperparameters used without optimization; no temporal/trend features considered; limited to cloud domain |
| 5 | RF + RL for Energy-Efficient VM Placement (Al-Rawi et al., 2025) | To optimize energy efficiency in cloud data centers through intelligent VM placement | Combined RF classification (using Self-Organizing Maps for VM sensitivity profiling) with Q-learning reinforcement learning for placement optimization | Up to 18.67% energy reduction over genetic algorithm baselines; maintained SLA compliance | Multi-model complexity increases deployment difficulty; Q-learning convergence can be slow; no IoT/edge validation; higher computational cost than single-model approaches |
| 6 | Enhanced RF for Intrusion Detection (Wang et al., 2024) | To improve network intrusion detection for high-dimensional and class-imbalanced security data | Bald Eagle Search (BES)-optimized Kernel PCA for dimensionality reduction combined with cost-sensitive RF for minority class detection | 5.59% improvement in classification accuracy; 11.32 seconds reduction in training time | Focused on security/intrusion detection, not resource allocation directly; performance depends on quality of dimensionality reduction; limited generalization to non-security domains |
| 7 | Systematic Literature Review: ML Resource Allocation (Patel et al., 2025) | To systematically survey ML-driven resource allocation across cloud, fog, and edge computing | SLR of 87 papers analyzing ML algorithms used for resource allocation; comparative evaluation across interpretability, accuracy, and deployment complexity | RF identified as 3rd most-used algorithm; highest-ranked in interpretability and deployment simplicity among all ML methods | Survey paper — no novel implementation; findings dependent on included papers' quality; limited to publications up to early 2025 |
| 8 | Comparative Review: AI/ML Cloud Resource Allocation (Hassan et al., 2025) | To systematically compare ten state-of-the-art AI/ML algorithms for cloud resource allocation | Comprehensive comparative evaluation across Deep RL, Neural Networks, Traditional ML, and Multi-Agent categories | Confirmed that hybrid architectures combining RF with optimization or RL consistently outperform single-method approaches | Review paper — no new experimental data; comparative criteria may not cover all real-world deployment factors; limited to cloud domain |

## Supporting Papers

| # | Paper Name | Objective | Methodology | Result | Limitations |
|---|---|---|---|---|---|
| P1 | RF for Big Data Pipeline Cost Prediction (Liu & Wang, 2025) | To predict resource costs in big data pipelines | Random Forest regression for workload-based cost estimation | Cost-aware resource planning validated on big data workloads | Limited to cost prediction; no allocation mechanism; preprint (not peer-reviewed) |
| P2 | Resource Utilization Survey: RF vs Others (Reddy & Kumar, 2025) | To benchmark RF against alternative ML models for resource utilization prediction | Survey comparing RF, SVM, ANN, and other models across multiple metrics | RF competitive with deep learning at significantly lower computational cost | Survey only — no new experiments; limited focus on real-time deployment |
| P3 | RF vs LSTM for CPU Efficiency (Mohammed & Salih, 2025) | To compare RF and LSTM for CPU efficiency optimization | Head-to-head comparison of RF and LSTM on CPU workload datasets | RF achieves comparable accuracy to LSTM with significantly lower training cost | Limited to CPU metric only; no multi-resource evaluation; single-domain testing |
| P4 | App-Oriented Workload Prediction Survey (Li et al., 2025) | To survey workload prediction techniques with application-oriented taxonomy | Literature survey categorizing prediction methods by application type | Comprehensive taxonomy of workload classification methods established | Survey paper; no novel model or experimental validation |
| P5 | ML Models for CPU/Memory/Network Proactive Scaling (Sharma et al., 2025) | To enable proactive multi-resource scaling using ML models | Multiple ML models (including RF) applied to CPU, memory, and network scaling | Demonstrated proactive scaling feasibility across multiple resource types | Limited real-world deployment validation; no hybrid optimization explored |
| P6 | GA-RF Hybrid VM Allocation (Nguyen & Park, 2023) | To apply GA-RF hybrid for VM allocation in cloud environments | GA-optimized RF for virtual machine placement decisions | Improved VM allocation efficiency over vanilla RF | Earlier and less comprehensive than [3]; limited dataset; no cross-domain testing |
| P7 | GRU/LSTM/RF for Dynamic Workload Prediction (Kim et al., 2024) | To compare deep learning and RF for dynamic workload prediction | Comparative study of GRU, LSTM, and RF on dynamic workload traces | RF competitive with deep models for workload prediction at lower cost | Prediction only — no allocation pipeline; limited to dynamic workloads |
| P8 | SLR: ML Resource Allocation — Springer (Müller & Weber, 2025) | To provide a comprehensive Springer-published systematic review | Systematic literature review with rigorous inclusion/exclusion criteria | Consolidated current state of ML-based resource allocation research | Survey — no new model; potentially overlapping scope with [7] |
| P9 | Lightweight RF for 5G IoT Security (Al-Hassani & Ibrahim, 2025) | To deploy lightweight RF for security detection on edge/IoT devices in 5G | Lightweight RF model designed for resource-constrained edge deployment | Demonstrated RF deployability on edge devices for real-time security | Focused on security, not resource allocation; 5G-specific |
| P10 | RF/SVM/LSTM for 5G Slice Intrusion Detection (Santos et al., 2025) | To compare RF, SVM, and LSTM for 5G network slice intrusion detection | Multi-model comparison across 5G network slicing security datasets | RF competitive with deep learning for intrusion detection tasks | Security-focused; not directly applicable to general resource allocation; domain-specific |
